#1 什么是网络流

定义 1.1（网络、源点汇点、容量）

定义网络 (G,S,T)(G,S,T)(G,S,T) 如下：

  • GGG 是一个边带非负权的有向图，SSS 和 TTT 是 GGG 中的节点；
  • SSS 被称为源点，TTT 被称为汇点；
  • 对于 GGG 中的边 (x→y,w)(x\to y,w)(x→y,w)，www 被称为改边的容量；

形象地：

对于一个带权有向图 GGG 及图上的两个点 SSS（源点）和 TTT（汇点），可以这样看待这张有向图：

  • 将点看作水池，边看作水管，则有向边 (x→y,w)(x\to y,w)(x→y,w) 表示单位时间内最多 www 单位的水可以通过该水管从水池 xxx 流向水池 yyy；

流就是让水从水源流到排水口而不爆水管的方案，流量就是方案中从排水口排走的水量。

2.1.2 EK 算法

直接模拟上述过程，可以得出求解网络流的朴素算法。

建完图后，用 bfs 来找增广路，并记录下每个点的前驱节点（令其入队的点）以便找到增广路上的边。如果 bfs 时访问到了汇点，那么就找到了一条增广路。

为了储存反向边，不妨令边的编号从 222 开始，并令编号为 iii 的边和编号为 i⊕1i\oplus 1i⊕1 的边互为反向边。这样实现起来会方便很多。

不断跑 bfs 找增广路去进行增广操作直到 SSS 和 TTT 不连通，即可求解出网络的一组最大流。该算法被称作 EK 算法，时间复杂度是 O((n+m)V)O((n+m)V)O((n+m)V) 的，其中 VVV 是最大流的大小，即值域。

2.1.3 dinic 算法

dinic 算法是 EK 算法的一种改进，其可以同时增广多条增广路。

其大体流程如下：

  • 每次增广前先跑一次 bfs，将节点按照距离 SSS 的最短路（边数）depudep_udepu​ 分层，增广时要求增广路相邻点的 depudep_udepu​ 恰好相差 111；
  • 接下来跑 dfs，对所有满足条件的增广路同时进行增广操作；
  • 不断重复上两步直至 bfs 时 SSS 不能到达 TTT，即不存在增广路；

伪代码如下：

int dfs(int u,int w) // u 表示当前节点的编号，w 表示流向当前点的流量，返回值为成功增广的流量
{
    int sum=0;
	遍历所有从 u 出发的边 i
	{
		int v=i 到达的节点;
		if(dep[v]==dep[u]+1)
		{
			int re=dfs(v,min(w,i 的容量));
			if(re!=0)
			{
				更新正向边和反向边的容量;
                w-=re;
				sum+=re;
			}
		}
	}
	return sum;
}
int dinic()
{
    int res=0;
    while(bfs 可以到达 汇点) res+=dfs(s,inf);
    return res;
}

此外，dinic 算法还有几个优化，分别是当前弧优化和断层优化：

  • 当前弧优化：把已经“榨干”（容量为 000）的边删掉；
  • 断层优化：把已经“榨干”的节点“删掉”（return 时若 sum==0 则令 dep[u]=0）；

一般默认会加上这两个优化。

dinic 的最劣复杂度为 O(n2m)O(n^2m)O(n2m)，证明考虑每次找到的增广路边数一定最小，故 depTdep_TdepT​ 递增；而单次 dfs 中每条增广路都会删掉至少一条边（当前弧优化）故最多有 mmm 条增广路，而每条增广路长度至多为 nnn。

并且 dinic 在容量均为 111 的网络上的复杂度为 O(mn)O(m\sqrt n)O(mn​)，具体证明及更多关于复杂度的信息可以看Dinic的几种复杂度 - myee - 博客园。

但由于网络流题目的图一般都有特殊性质，故加上优化后的 dinic 常常十分快速，基本不会被卡，在 OI 中最常用。

求解网络流还有 ISAP 和 HLPP，由于太过复杂且 OI 中不考，故不再介绍。

#3 最小费用最大流

在最小费用最大流问题中，每条边新增了一个单位代价 wiw_iwi​，表示这条边流过 111 流量需要 wiw_iwi​ 的代价，需要求解一组总代价最小的最大流。

也可以使用 dinic 算法解决，每次增广单位代价最小的增广路即可，正确性显然。建边时反向边代价为 −wi-w_i−wi​，而且 bfs 需要换成 spfa，且增广时要求增广路相邻点的 depudep_udepu​ 恰好相差它们之间的边的代价。

值得注意的是，由于代价可能为 000，所以 dfs 时需要防止重复访问点。

#4 有向图最小割

定义 4.1（有向图的割）

对于一个带权有向图 GGG 及图上的两个点 SSS 和 TTT，定义一组 SSS 到 TTT 的割 CCC 如下：

  • C∈E(G)C\in E(G)C∈E(G)；
  • 删掉 CCC 中的边后 SSS 不能到达 TTT；

定义割 CCC 的大小 ∣C∣|C|∣C∣ 为 ∑(xi→yi,wi)∈Cwi\sum\limits_{(x_i\to y_i,w_i)\in C} w_i(xi​→yi​,wi​)∈C∑​wi​，即割中的边权和。

定义 4.2（有向图的最小割）

对于一个带权有向图 GGG 及图上的两个点 SSS 和 TTT，定义其最小割 MinCut(G,S,T)\text{MinCut}(G,S,T)MinCut(G,S,T) 为 ∣C∣|C|∣C∣ 最小的一组割 CCC。

定理 4.1（最大流等于最小割）

对于一个带权有向图 GGG 及图上的两个点 SSS 和 TTT，有：

∣MaxFlow(G,S,T)∣=∣MinCut(G,S,T)∣|\text{MaxFlow}(G,S,T)|=|\text{MinCut}(G,S,T)| ∣MaxFlow(G,S,T)∣=∣MinCut(G,S,T)∣

证明很简单，首先由于跑完最大流算法后 SSS 不能到达 TTT，故 ∣MaxFlow(G,S,T)∣≥∣MinCut(G,S,T)∣|\text{MaxFlow}(G,S,T)|\ge|\text{MinCut}(G,S,T)|∣MaxFlow(G,S,T)∣≥∣MinCut(G,S,T)∣，又显然 ∣MaxFlow(G,S,T)∣≤∣MinCut(G,S,T)∣|\text{MaxFlow}(G,S,T)|\le |\text{MinCut}(G,S,T)|∣MaxFlow(G,S,T)∣≤∣MinCut(G,S,T)∣（最大流中满流的边集构成割），故得证。

更详细的证明可以自行上网寻找。

并且根据一组最大流 FFF，可以简单地构造出最小割：

  • 令每条边的剩余容量 cic_ici​ 为 wi−fiw_i-f_iwi​−fi​；
  • 从 SSS 出发，只经过 ci>0c_i>0ci​>0 的边，求出能到达的点集 AAA；
  • 一组最小割 CCC 即为 {(x→y,w)∈E(G)∣x∈A,y∈(V(G)−A)}\{(x\to y,w)\in E(G)|x\in A,y\in (V(G)-A)\}{(x→y,w)∈E(G)∣x∈A,y∈(V(G)−A)}，即一端在 AAA 中的边的集合；

#5 有上下界网络流

5.1 有上下界循环可行流

nnn 个点，mmm 条有向边，每条边有一个流量下界 lil_ili​ 和流量上界 rir_iri​，求该图的一个循环可行流，即你需要构造 mmm 个变量 fif_ifi​ 满足：

  • 流量合法：∀i,li≤fi≤ri\forall i,l_i\le f_i\le r_i∀i,li​≤fi​≤ri​；
  • 流量平衡：对于每个点 iii，设 IiI_iIi​ 为 iii 的入边集合，UiU_iUi​ 为 iii 的出边集合，那么有：∑j∈Iifj=∑j∈Uifj\sum\limits_{j\in I_i} f_j=\sum\limits_{j\in U_i}f_jj∈Ii​∑​fj​=j∈Ui​∑​fj​；

考虑每条边先钦定流过 lil_ili​ 的流量，转化为每条边只有流量上界 ri−lir_i-l_iri​−li​，那么流量有可能不平衡。

考虑用点与源点 SSS 或汇点 TTT 的连边刻画每个点的额外流量。记 bi=∑j∈Iilj−∑j∈Uiljb_i=\sum\limits_{j\in I_i} l_j-\sum\limits_{j\in U_i}l_jbi​=j∈Ii​∑​lj​−j∈Ui​∑​lj​，即每个点进入的流量减出去的流量，那么：

  • 若 bi<0b_i<0bi​<0，连接边 (i→T,−bi)(i\to T,-b_i)(i→T,−bi​)；
  • 若 bi>0b_i>0bi​>0，连接边 (S→i,bi)(S\to i,b_i)(S→i,bi​)；

原先的边 iii 直接连 (xi→yi,ri−li)(x_i\to y_i,r_i-l_i)(xi​→yi​,ri​−li​)。

跑最大流，若所有 (i→T,−bi)(i\to T,-b_i)(i→T,−bi​) 和 (S→i,bi)(S\to i,b_i)(S→i,bi​) 均满流，则有解，否则无解。设跑完后第 iii 条边的流量为 cic_ici​，则第 iii 条边的流量为 li+cil_i+c_ili​+ci​。

5.2 有上下界有源汇最大流

考虑将其转化为循环流来消去上下界，设源点为 sss，汇点为 ttt，则不难发现相当于从 sss 出发的流量等于到达 ttt 的流量，连边 (t→s,[0,∞])(t\to s,[0,\infin])(t→s,[0,∞])。

跑出循环流后，记 resresres 为 (t→s,[0,∞])(t\to s,[0,\infin])(t→s,[0,∞]) 的。接下来把 (t→s,[0,∞])(t\to s,[0,\infin])(t→s,[0,∞]) 去掉，设第 iii 条边在循环流中流过了 cic_ici​ 流量，则连接边 (xi→yi)(x_i\to y_i)(xi​→yi​) 时：

  • 正向边流量为 ri−cir_i-c_iri​−ci​；
  • 反向边流量为 ci−lic_i-l_ici​−li​；

5.3 有上下界有源汇最小流

最大流是尽可能从 sss 流到 ttt，那么最小流就是尽可能从 ttt 退回 sss。

跑出 ttt 到 sss 的最大流 rfrfrf，答案即为 res−rfres-rfres−rf。
