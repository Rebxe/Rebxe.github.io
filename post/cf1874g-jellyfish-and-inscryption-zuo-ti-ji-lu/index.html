CF1874G Jellyfish and Inscryption 做题记录

发布于 2025-11-19 | 标签: 结论 、 DP 、 做题记录 | 12分钟 | 2289字数

给定一张 nnn 个点 mmm 条边的 DAG，每条边 ui→viu_i\to v_iui​→vi​ 都满足 ui<viu_i<v_iui​<vi​。

点有五种类型：

  1. 无特殊事件；
  2. 走到这里会获得一个二元组 (a,b)(a,b)(a,b)；
  3. 走到这里可以选择一个之前获得的二元组，将其的 aaa 增加 xxx；
  4. 走到这里可以选择一个之前获得的二元组，将其的 bbb 增加 xxx；
  5. 走到这里会获得一个数 www；

你要从点 111 走到点 nnn，保证点 111 和点 nnn 是 111 类点。

走到点 nnn 时可以选择一个之前获得的二元组，将其的 bbb 乘上 10910^9109。

最大化最终获得的所有二元组的 a×ba\times ba×b 的和加上获得的数 www 的和。

1≤n≤2001\le n\le 2001≤n≤200，1≤m≤min⁡(n(n−1)2,2000)1\le m\le \min(\frac{n(n-1)}{2},2000)1≤m≤min(2n(n−1)​,2000)，1≤a,b,x≤2001\le a,b,x\le 2001≤a,b,x≤200，1≤w≤1061\le w\le 10^61≤w≤106。

下文设 V=200V=200V=200。

这个最后乘 10910^9109 的操作其实相当于先最大化 a×ba\times ba×b 的最大值，再最大化 ∑a×b\sum a\times b∑a×b。

考虑枚举这个最大的 (a,b)(a,b)(a,b) 是点 uuu 处加入的，那么 u→nu\to nu→n 路径上的 (+x,0)(+x,0)(+x,0) 和 (0,+x)(0,+x)(0,+x) 一定都操作到它的头上。这部分的贡献可以简单 dp 算出：

  • 设 fu,a=(b,w)f_{u,a}=(b,w)fu,a​=(b,w) 表示从 uuu 出发走到 nnn 且 (+x,0)(+x,0)(+x,0) 操作的和为 (+a,0)(+a,0)(+a,0) 时，(0,+x)(0,+x)(0,+x) 操作的和最大为 (0,+b)(0,+b)(0,+b)，在此前提下其余贡献（+w+w+w 和其它二元组的 +a×b+a\times b+a×b）最大为 www；

现在问题在于计算 1→u1\to u1→u 的最大贡献。

不难发现，点 uuu 的二元组 (au,bu)(a_u,b_u)(au​,bu​) 若在点 vvv 处被操作了 (+xv,0)(+x_v,0)(+xv​,0)，那么 u→vu\to vu→v 路径上的所有 (+xv,0)(+x_v,0)(+xv​,0) 都应该操作到 uuu 头上。所以一个二元组一旦不再成为 (+x,0)(+x,0)(+x,0) 的目标，那么它之后就再也不会成为 (+x,0)(+x,0)(+x,0) 的目标了。(0,+x)(0,+x)(0,+x) 同理。

所以现在有一个 O(n3V2)O(n^3V^2)O(n3V2) 的 dp：

  • 设 fu,a,b,0/1f_{u,a,b,0/1}fu,a,b,0/1​ 表示从 111 走到了 uuu，当前 (0,+x)(0,+x)(0,+x) 操作的目标二元组是 (a,∗)(a,*)(a,∗)，当前 (+x,0)(+x,0)(+x,0) 操作的目标二元组是 (∗,b)(*,b)(∗,b)，这两个二元组是否来自同一个点（是否是二元组 (a,b)(a,b)(a,b)）；

转移只需要考虑新加入的二元组是否会干掉 (a,∗)(a,*)(a,∗) 和 (∗,b)(*,b)(∗,b) 或者两个都干掉。

进一步，如果 a>Va>Va>V 且 b>Vb>Vb>V，那么该状态一定是 fu,a,b,1f_{u,a,b,1}fu,a,b,1​，即对应二元组为 (a,b)(a,b)(a,b)。那么 u→nu\to nu→n 路径上的 (+x,0)(+x,0)(+x,0) 和 (0,+x)(0,+x)(0,+x) 一定都会操作到它头上，所以这个二元组可以看作是最后 a×ba\times ba×b 最大的那个，即我们在钦定 (au,bu)(a_u,b_u)(au​,bu​) 作为 a×ba\times ba×b 最大的那个二元组计算答案时已经算过这种情况。

所以有 min⁡(a,b)≤V\min(a,b)\le Vmin(a,b)≤V，而注意到 aaa 和 bbb 在转移过程中不会减少，所以可以把一维开到 VVV。不妨假定 a≤Va\le Va≤V，这样交换 a,ba,ba,b 再做一次即可。

那么这个 dp 被优化到了 O(n2V2)O(n^2V^2)O(n2V2)。

继续。考虑二元组的贡献形如 (a+∑x)×(b+∑x)(a+\sum x)\times (b+\sum x)(a+∑x)×(b+∑x)，我们这个 dp 算贡献的方式是加入一个元素就算一次它和当前另一边的贡献，所以两边的总和都要记录。

注意到根据之前的性质，有 a+∑x≤Va+\sum x\le Va+∑x≤V，那么不妨直接钦定左边 a+∑xa+\sum xa+∑x 的值，将算贡献的方式改为每次加入一个右边的元素 zzz（bbb 或 (0,+x)(0,+x)(0,+x)）时贡献 (a+∑x)×z(a+\sum x)\times z(a+∑x)×z，相当于将 (+x,0)(+x,0)(+x,0) 操作“预支”出去。再根据一个二元组 (a,b)(a,b)(a,b) 一定是在路径的一个区间内成为 (+x,0)(+x,0)(+x,0) 操作的目标，故每个时刻只有最多一个二元组的债还未偿还。

那么有 dp：

  • 设 fu,a,kf_{u,a,k}fu,a,k​ 表示从 111 走到了 uuu，当前 (a+∑x)(a+\sum x)(a+∑x) 为 aaa，还欠了 kkk 的债；

转移是简单的，时间复杂度 O((n+m)V2)O((n+m)V^2)O((n+m)V2)。

注意要特判没有 111 类点的情况，代码如下：

#include <iostream>
#include <cstdio>
#include <vector>

using namespace std;

typedef long long ll;

const int S=205,MUL=1000000000;
const ll inf=1e17;

int n,m;
int tp[S],a[S],b[S];
vector<int> g[S],rg[S];
ll f[S][S][S],res[S];
pair<ll,ll> h[S][S*S];

template<typename T>
inline void tmax(T &x,T y){x=max(x,y);}

inline void doit()
{
	for(int i=1;i<=n;i++)
		for(int j=0;j<=S-3;j++)
			for(int k=0;k<=S-3;k++) f[i][j][k]=-inf;
	f[1][0][0]=0;
	for(int i=1;i<=n;i++)
		for(int v:g[i])
			for(int j=0;j<=S-3;j++)
				for(int k=0;k<=S-3;k++)
				{
					ll u=f[i][j][k];
					if(u<0) continue;
					if(tp[v]==0) tmax(f[v][j][k],u);
					if(tp[v]==1)
					{
						tmax(f[v][max(j,a[v])][k],u+a[v]*b[v]);
						if(k==0)
						{
							for(int x=a[v];x<=S-3;x++)
								tmax(f[v][max(j,x)][x-a[v]],u+x*b[v]);
						}
					}
					if(tp[v]==2) tmax(f[v][j][max(0,k-a[v])],u);
					if(tp[v]==3) tmax(f[v][j][k],u+j*b[v]);
					if(tp[v]==4) tmax(f[v][j][k],u+a[v]);
				}
	for(int i=1;i<=n;i++)
		for(int j=0;j<=S-3;j++) tmax(res[i],f[i][j][0]);
}

int main()
{
	ios::sync_with_stdio(false),cin.tie(0),cout.tie(0);
	cin>>n>>m;
	for(int i=1;i<=n;i++)
	{
		cin>>tp[i];
		if(tp[i]==0) continue;
		if(tp[i]==1) cin>>a[i]>>b[i];
		if(tp[i]==2) cin>>a[i];
		if(tp[i]==3) cin>>b[i];
		if(tp[i]==4) cin>>a[i],b[i]=a[i];
	}
	for(int i=1;i<=m;i++)
	{
		int x,y;
		cin>>x>>y;
		g[x].push_back(y);
		rg[y].push_back(x);
	}
	for(int i=1;i<=n;i++) res[i]=-inf;
	doit();
	for(int i=1;i<=n;i++)
	{
		swap(a[i],b[i]);
		if(tp[i]==2||tp[i]==3) tp[i]=2+3-tp[i];
	}
	doit();
	for(int i=1;i<=n;i++)
		for(int j=0;j<=S*S-3;j++) h[i][j]=make_pair(-inf,-inf);
	h[n][0]=make_pair(0,0);
	for(int i=n;i>=1;i--)
		for(int v:rg[i])
			for(int j=0;j<=S*S-3;j++)
			{
				pair<ll,ll> u=h[i][j];
				if(u.first<0) continue;
				if(tp[v]==0) tmax(h[v][j],u);
				if(tp[v]==1)
					tmax(h[v][j],make_pair(u.first,u.second+a[v]*b[v]));
				if(tp[v]==2&&j+a[v]<=S*S-3)
					tmax(h[v][j+a[v]],u);
				if(tp[v]==3)
					tmax(h[v][j],make_pair(u.first+b[v],u.second));
				if(tp[v]==4)
					tmax(h[v][j],make_pair(u.first,u.second+a[v]));
			}
	ll ans=0;
	for(int i=1;i<=n;i++)
		if(tp[i]==1)
			for(int j=0;j<=S*S-3;j++)
			{
				if(h[i][j].first<0) continue;
				ll pre=res[i]+h[i][j].second-a[i]*b[i]*2;
				ll val=(a[i]+j)*(b[i]+h[i][j].first);
				tmax(ans,pre+val*MUL);
			}
	for(int i=0;i<=S-3;i++) tmax(ans,f[n][i][0]);
	cout<<ans<<'\n';
	return 0;
}
