P3808 AC 自动机（简单版）

给定 nnn 个模式串 tit_iti​ 和一个文本串 sss，求出 sss 中有多少个 tit_iti​ 出现了。

考虑先将所有模式串插入到一棵 Trie 树中，然后在上面跑（不停从点 uuu 走到 uuu 的边权为 sis_isi​ 的儿子，即令 uuu 变为 sonu,sison_{u,s_i}sonu,si​​）。这样若当前点 uuu 为某个 tit_iti​ 对应的点（可以在插入时给这个点打标记），则说明 tit_iti​ 在 sss 中出现了。

但是这样不停跑可能会出现这样的情况：当前位于点 uuu，而 sonu,si=0son_{u,s_i}=0sonu,si​​=0 即不存在对应的儿子，此时我们无法直接走到 sonu,sison_{u,s_i}sonu,si​​。不妨称这种情况为“失配”。

解决办法很简单，假设第一次出现这种情况时跑完了 s[1,i]s_{[1,i]}s[1,i]​，当前在点 uuu 且 sonu,si+1=0son_{u,s_{i+1}}=0sonu,si+1​​=0。则此时 uuu 对应的字符串是 s[1,i]s_{[1,i]}s[1,i]​。为了能继续往下跑，我们可以跳到一个 Trie 树上的节点 vvv，使得 vvv 代表的字符串是 s[1,i]s_{[1,i]}s[1,i]​ 的后缀，且 v=uv\not=uv​=u：

如果这样的 vvv 有很多个，那么跳到最深（代表的字符串最长）的那个一定是最优的（因为从它出发能跳到其它合法的 vvv）。并且由于 Trie 中每个点的字符串两两不同，故最深的 vvv 一定只有一个。

不妨令 failufail_ufailu​ 为最深的这样的 vvv，即对应字符串是点 uuu 对应字符串的后缀且深度最大的点 vvv（需满足 v=uv\not=uv​=u）。

考虑 failfailfail 的求解。注意到 failufail_ufailu​ 的深度一定小于 uuu 的深度，故可以使用 bfs 求解。先将 fail0fail_0fail0​ 以及所有 000 的儿子的 failfailfail 设为 000。对于一个点 uuu 及 v=sonu,i=0v=son_{u,i}\not=0v=sonu,i​​=0：

  • 令 p=failup=fail_up=failu​ 并不断令 p=failpp=fail_pp=failp​，直到：
    • p=0p=0p=0，则令 failv=0fail_v=0failv​=0；
    • sonp,i=0son_{p,i}\not=0sonp,i​​=0，则令 failv=sonp,ifail_v=son_{p,i}failv​=sonp,i​；

暴力跳太慢了，不妨在 sonu,i=0son_{u,i}=0sonu,i​=0 时令 sonu,i=sonfailu,ison_{u,i}=son_{fail_u,i}sonu,i​=sonfailu​,i​，这样就不用暴力跳了：

  • 若 sonu,i=0son_{u,i}\not=0sonu,i​​=0，则令 failsonu,i=sonfailu,ifail_{son_{u,i}}=son_{fail_u,i}failsonu,i​​=sonfailu​,i​；

即 sonu,ison_{u,i}sonu,i​ 实际上变成了：

  • 从 uuu 出发，不断令 u=failuu=fail_uu=failu​ 直到边权为 iii 的儿子 vvv 存在，sonu,ison_{u,i}sonu,i​ 就为这个 vvv；

求 failfailfail 数组的代码如下：

inline void build()
{
	queue<int> q; 
	fail[0]=0;
	for(int i=0;i<26;i++)
	{
		int v=son[0][i];
		if(v!=0) fail[v]=0,q.push(v);
	}
	while(!q.empty())
	{
		int u=q.front();
		q.pop();
		for(int i=0;i<26;i++)
		{
			int v=son[u][i];
			if(v!=0) fail[v]=son[fail[u]][i],q.push(v);
			else son[u][i]=son[fail[u]][i];
		}
	}
}

对于例题，不难发现当在点 uuu 时，实际上能匹配 u,failu,failfailu,…u,fail_u,fail_{fail_u},\dotsu,failu​,failfailu​​,… 对应的字符串，即不断跳 failfailfail 经过的点都能匹配到。故还需记录一个点是否被跳过了，如果经过跳过的点就停止跳 failfailfail：

inline int calc(char* s,int n) // sum[u] 表示有多少个文本串等于 u 表示的字符串
{
	int u=0,res=0;
	for(int i=1;i<=n;i++)
	{
		int id=s[i]-'a'; 
		u=son[u][id];
		int v=u;
		while(v!=0&&sum[v]!=-1) // 跳到根节点或者跳到过就不用跳 
		{
			res+=sum[v]; // 计算贡献 
			sum[v]=-1;   // 标记跳到过了 
			v=fail[v];   // 继续跳 fail 
		}
	}
	return res;
}

实际上，由于每个点只有一个 failfailfail，且不断跳 failfailfail 必定停留在点 000，故 failfailfail 实际上是一棵根为 000，包含所有 Trie 中节点的树，即 fail 树。
